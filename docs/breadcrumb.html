The graphical control element Breadcrumbs or breadcrumb trail is a navigation aid used in user interfaces. It allows users to keep track of their locations within programs or documents. Breadcrumbs typically appear horizontally across the top of a Web page, often below title bars or headers. They provide links back to each previous page the user navigated through to get to the current page or—in hierarchical site structures—the parent pages of the current one. Breadcrumbs provide a trail for the user to follow back to the starting or entry point.

In our site breadcrumb is used to indecate the steps involved in booking process.

For this reason we consider breadcrumb element as a step listing item

Markup

Breadcrumb should not be considered as <nav> tag, even if it is consisted by a set of links. <nav> element is intended only for major block of navigation links. Browsers, such as screen readers for disabled users, can use this element to determine whether to omit the initial rendering of this content.

Breadcrumb should not be considered as <ul> element. The HTML is not semantic. Coding a breadcrumb as an unordered list makes it seem like the items are on the same hierarchical level, like it's a set of items in a row. The items in a breadcrumb are not, they represent a path, with each link being a "child" of the last.

Breadcrumbs are in a very specific order and so using an "ordered" list seems appropriate. I'm somehow not totally satisfied by that though. Just because a list is 1, 2, 3, etc, doesn't really represent layers of hierarchy.
